## Detailed description

```
FeedBat is your trusty companion for finding feeds in the wild. 🦇

Stop hunting through page source for the subscribe link. FeedBat automatically
detects RSS, Atom, and JSON feeds on whatever page you're viewing and shows them
in a clean popup — ready to copy into your favorite RSS reader or open in a new
tab with one click.

WHAT IT DOES
• Automatically detects RSS, Atom, JSON Feed, and RDF feeds on any site
• Shows a badge with the number of feeds found
• One-click copy of any feed URL to your clipboard
• One-click open of any feed in a new tab
• "Check common paths" button probes standard locations (/feed, /rss.xml,
  /atom.xml, …) when a site doesn't advertise its feed
• Finds YouTube channels' hidden RSS feeds — on video, Shorts, and channel
  pages — even though YouTube never links to them

PRIVACY FIRST
FeedBat has no servers, no analytics, and no tracking. Page analysis happens
entirely in your browser, and nothing you browse is ever sent to the developer.
Full policy: https://schmug.github.io/FeedBat/privacy.html

OPEN SOURCE
FeedBat is MIT licensed. Source: https://github.com/schmug/FeedBat
Feed detection logic is adapted from the excellent feedsearch-crawler library
by DBeath (https://github.com/DBeath/feedsearch-crawler).
```

## Permission justifications (Dashboard → "Privacy practices")

> Reviewer-facing. Be literal and specific.

**`activeTab`**
```
Used to read the DOM of the page the user is currently viewing (link tags, anchor
links, meta tags) to detect feeds, and to send HEAD requests to that same site's
common feed paths when the user clicks "Check common paths". Access is only
granted when the user opens the FeedBat popup.
```

**`storage`**
```
Stores the user's four on/off preferences (auto-detect, show potential feeds,
show badge count, auto-probe common paths) via chrome.storage.sync. No browsing
data or personal information is stored.
```

**Host access / content script on all sites (`content_scripts` matches `<all_urls>`)**
```
Feed detection must be able to run on whatever website the user chooses to
visit, since any site may publish a feed. The content script only reads page
markup to locate feed URLs; it never modifies pages, injects content, or
transmits page data anywhere. No remote host permissions are requested.
```

**Remote code:** None. All code is contained in the package; no remote/eval'd code.
